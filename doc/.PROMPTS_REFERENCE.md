# RHIS Configuration Prompts Reference

## Overview
This document lists all configuration prompts added to support the enhanced Satellite and IdM kickstart features for comprehensive infrastructure automation.

## Prompts by Category

### Satellite Provisioning Configuration
Used by: Menu choices 3, 4, 5, 7, 9 (any Satellite deployment)

| Variable | Default | Description |
|----------|---------|-------------|
| `SAT_PROVISIONING_SUBNET` | `10.168.0.0` | Satellite provisioning network subnet (CIDR or address) |
| `SAT_PROVISIONING_NETMASK` | `${NETMASK}` | Satellite provisioning network netmask |
| `SAT_PROVISIONING_GW` | `10.168.0.1` | Satellite provisioning network gateway |
| `SAT_PROVISIONING_DHCP_START` | `10.168.130.1` | DHCP pool start IP address |
| `SAT_PROVISIONING_DHCP_END` | `10.168.255.254` | DHCP pool end IP address |
| `SAT_PROVISIONING_DNS_PRIMARY` | `${SAT_IP}` | Primary DNS for provisioning subnet (defaults to Satellite IP) |
| `SAT_PROVISIONING_DNS_SECONDARY` | `8.8.8.8` | Secondary DNS for provisioning subnet |
| `SAT_DNS_ZONE` | `${DOMAIN}` | Forward DNS zone served by Satellite |
| `SAT_DNS_REVERSE_ZONE` | derived from provisioning subnet | Reverse DNS zone served by Satellite |
| `SAT_FIREWALLD_INTERFACE` | `eth1` | Internal interface used for Satellite infra services |
| `SAT_FIREWALLD_ZONE` | `public` | Firewalld zone for Satellite infra services |
| `SAT_FIREWALLD_SERVICES_JSON` | `["ssh","http","https"]` | Services opened in the selected firewalld zone |

**Usage in Kickstart:**
- 5.1.6 Configure Provisioning Subnet
- Hammer CLI: `hammer subnet create` command options

### Satellite Repository Configuration
Used by: Menu choices 4, 5, 7, 9 (any Satellite deployment)

| Variable | Default | Description |
|----------|---------|-------------|
| `SAT_RHEL10_BASEOS_REPO` | `rhel-10-for-x86_64-baseos-rpms` | RHEL 10 BaseOS repository name (must exist in Satellite) |
| `SAT_RHEL10_APPSTREAM_REPO` | `rhel-10-for-x86_64-appstream-rpms` | RHEL 10 AppStream repository name (must exist in Satellite) |
| `SAT_RHEL9_BASEOS_REPO` | `rhel-9-for-x86_64-baseos-rpms` | RHEL 9 BaseOS repository name (must exist in Satellite) |
| `SAT_RHEL9_APPSTREAM_REPO` | `rhel-9-for-x86_64-appstream-rpms` | RHEL 9 AppStream repository name (must exist in Satellite) |

**Usage in Kickstart:**
- 5.1.3 Create Content Views for RHEL 10 & 9
- Hammer CLI: `hammer content-view add-repository` command

**Notes:**
- Repository names must match exactly those synced in Satellite
- Recommended naming format: `rhel-VERSION-for-ARCH-STREAM-rpms`
- If repository doesn't exist, hammer command will fail with clear error

### IdM User Groups & Access Control
Used by: Menu choices 4, 5, 7, 10 (any IdM deployment)

| Variable | Default | Description |
|----------|---------|-------------|
| `IDM_ADMINS_GROUP` | `rhis-admins` | IdM group name for infrastructure administrators |
| `IDM_CONTENT_MANAGERS_GROUP` | `content-managers` | IdM group name for Satellite/content managers |
| `IDM_AUTOMATION_ENGINEERS_GROUP` | `automation-engineers` | IdM group name for AAP/Ansible automation users |
| `IDM_SYSTEM_SERVICES_GROUP` | `system-services` | IdM group name for system service accounts |
| `IDM_ENABLE_HBAC_RULES` | `1` | Enable Host-Based Access Control (HBAC) rules (1=yes, 0=no) |
| `IDM_ENABLE_SUDO_RULES` | `1` | Enable SUDO delegation rules (1=yes, 0=no) |

**Usage in Kickstart:**
- 5.1.1 Create IdM User Groups
- 5.1.2 Create IdM Users for RHIS Components (adds users to groups)
- 5.1.4 Configure Host-Based Access Control (HBAC) - uses admin group
- 5.1.5 Configure SUDO Rules - uses admin and content-manager groups
- IPA CLI: `ipa group-add`, `ipa group-add-member`, `ipa hbacrule-add`, `ipa sudorule-add`

**Notes:**
- Group names are user-customizable for organizational preferences
- HBAC and SUDO rules can be disabled entirely for simpler deployments
- When disabled, users still created but no access control policies applied
- All rules use standard RHIS role names for clear semantics

## Prompt Flow in Interactive Mode

### First Run (Bootstrap)
1. Global configuration (domain, network, credentials)
2. Satellite configuration (IP, hostname, org, location)
3. AAP configuration (IP, hostname, bundle URL)
4. IdM configuration (IP, hostname, passwords)
5. **NEW: Satellite Provisioning** (subnet, DHCP, DNS)
6. **NEW: Satellite Repositories** (BaseOS/AppStream repo names for RHEL 10 & 9)
7. **NEW: IdM Access Control** (group names, HBAC/SUDO enable)

All values stored encrypted in: `${ANSIBLE_ENV_FILE}` (default: `$HOME/.ansible/conf/env.yml`)

### Reconfigure Mode (`--reconfigure` flag)
Users re-prompted for all values above with current defaults shown. Useful for:
- Changing provisioning subnet
- Customizing group names
- Enabling/disabling HBAC/SUDO rules
- Updating repository names if repos were renamed in Satellite

### Headless Mode (CI/CD)
All variables can be pre-set via:
```bash
export SAT_PROVISIONING_SUBNET="192.168.100.0"
export IDM_ADMINS_GROUP="infrastructure-team"
export IDM_ENABLE_HBAC_RULES="0"
./rhis_installer.sh --non-interactive --menu-choice 9
```

## Validation & Defaults

### Satellite Provisioning
- All DHCP IPs validated as valid IP addresses
- Subnet must be in CIDR notation or valid IP
- DNS servers validated as valid IPs
- If any value empty, defaults used automatically

### Repositories
- Repository names used as-is in hammer commands
- If repo doesn't exist in Satellite, kickstart notifies but continues
- Content view creation retried on each run (idempotent)

### IdM Groups & Rules
- Group names must be valid LDAP group names (alphanumeric, hyphens)
- HBAC/SUDO enable values: `1`/`yes`/`true` = enabled, `0`/`no`/`false` = disabled
- Rules apply to custom group names (you must use custom group names in policies)
- All IAM operations wrapped with error handling (skip if already exist)

## Implementation Details

### Prompt Addition Locations
- **Interactive prompts:** `prompt_all_env_options_once()` function (lines ~7300-7400)
- **Satellite kickstart usage:** `generate_satellite_618_kickstart()` function (lines ~8500-8700)
- **IdM kickstart usage:** `generate_idm_kickstart()` function (lines ~10000-10100)

### Variable Normalization
- `normalize_shared_env_vars()` function validates all values after assignment
- Defaults applied via `prompt_with_default()` function parameter
- Empty values fall back to hardcoded defaults

### Encryption
- All variables stored in Ansible vault encrypted file
- Vault password required only on first run
- Subsequent runs use cached password from system keyring (if available)

## Troubleshooting

### "Repository XYZ not found" Error
**Cause:** Repository name doesn't match what's in Satellite
**Solution:** 
1. Check Satellite UI for exact repo names
2. Run `--reconfigure` and update `SAT_RHEL*_*_REPO` variables
3. Ensure repos are enabled/synced in Satellite

### HBAC Rules Not Working
**Cause:** `IDM_ENABLE_HBAC_RULES` set to `0`, or users not added to groups
**Solution:**
1. Enable: `export IDM_ENABLE_HBAC_RULES="1"`
2. Verify users are in correct IdM group: `ipa group-member <group_name>`
3. Verify HBAC rule exists: `ipa hbacrule-find`

### Custom Group Names Not Appearing
**Cause:** Group names not propagated to all rules
**Solution:**
1. Verify groups exist: `ipa group-find`
2. Reset IdM config: `--reconfigure`
3. Manually add missing membership: `ipa group-add-member <group> --users=<user>`
